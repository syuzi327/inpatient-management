# 入院タスク管理アプリ

病棟業務向けの入院患者タスク管理Webアプリです。患者ごとにクリニカルパスのタスクを自動生成・管理できます。

**URL**: https://syuzi327.github.io/inpatient-management/

---

## 主な機能

- 患者登録時にテンプレートからタスクを自動生成
- 入院前・日別・退院後のタスクを日付ごとに表示
- 各タスクのWhy/Howメモ（タップで詳細表示）
- 退院日は未定のまま登録可能、後から変更可能
- テンプレートはGoogleスプレッドシートで管理
- データはブラウザのlocalStorageに保存（サーバー不要）
- ダークテーマ・モバイルファースト

---

## Googleスプレッドシート設定

### 公開設定

1. スプレッドシートを開く
2. 「ファイル」→「共有」→「ウェブに公開」
3. 「ドキュメント全体」・「CSV」を選択して公開
4. 発行されたURLを `index.html` 内の `SHEET_URL` に設定

### シート構成

#### indexシート（1枚目）

テンプレートの一覧を管理します。

| 列 | 内容 |
|----|------|
| `template_key` | 英数字・アンダーバーのみのID |
| `template_name` | アプリ上の表示名 |
| `gid` | シートのGID（URLの `#gid=XXXX` の数値） |
| `memo` | 詳細ページに表示する疾患メモ（改行可） |

#### テンプレートシート（疾患ごと）

ヘッダー行: `type,day,title,cat,why,how`

| 列 | 内容 |
|----|------|
| `type` | `pre`（入院前）/ `day`（日付あり）/ `post`（退院後） |
| `day` | typeがdayの場合のみ記入（下表参照） |
| `title` | タスク名 |
| `cat` | カテゴリ（下表参照） |
| `why` | なぜ行うか（タスク詳細に表示） |
| `how` | やり方・コツ（タスク詳細に表示） |

#### day値の意味

| day | 表示 |
|-----|------|
| `0` | 入院前日 |
| `1` | 入院日 |
| `2` | 第2病日 |
| `3` | 第3病日（以降同様） |
| `-1` | 退院前日 |
| `-0.1` | 退院当日 |

#### カテゴリ一覧（cat列）

| 値 | 色 |
|----|----|
| `検査` | 青 |
| `処置` | 赤 |
| `投薬` | オレンジ |
| `看護` | ティール |
| `書類` | 黄 |
| `説明` | ピンク |
| `食事` | 紫 |
| `リハビリ` | 緑 |
| `連絡` | インディゴ |
| `その他` | グレー |

## ローカル開発

```bash
# リポジトリをクローン
git clone https://github.com/syuzi327/inpatient-management.git
cd inpatient-management

# index.html をブラウザで直接開くだけで動作します
```

## デプロイ

GitHub Pagesを使用しています。`main`ブランチにpushすると自動でデプロイされます。

git push
